## 背景

v1.0 上架版策略是"全功能免费解锁、无试用限制"。v1.1.1 是 v1.1 系列的第二个子版本（v1.1.0 稳定性之后），目标是**把手势语义从"选风格"颠覆为"横滑即切台"**，并修复 v1.0 用户反馈的 Evolve 体感问题。

本次 brainstorming 基于四个输入交叉：
1. 老鱼的真实听感反馈（Evolve 听感差/有时切风格）
2. v1.1 规划原文的 7 条 v1.1.1 抓手
3. 代码现状（`AppState.swift` / `SpectrumCarouselView.swift` / `PlayControlView.swift` / `SettingsView.swift`）
4. v1.1.0 已完成（30s ring buffer + watchdog + artwork 等 invisible fixes，并明确撤回 25min 自动换台）

## 核心原则（必须同时满足）

1. **主页 + 沉浸页 UI 零视觉改动** — v1.0 已上架且老鱼认可视觉，按钮/配色/布局/字体/尺寸一个像素不动。只换底层 binding 和 handler。
2. **不破坏沉浸感** — 稳定性逻辑走后台不可见方案，任何自动行为不打断用户聆听。
3. **数据可逆** — `UserDefaults` / `Keychain` schema 改动必须保留旧 key 一个版本，0 迁移成本优先。
4. **Git 粒度可逆** — 每个独立抓手单独 commit，出问题能 `git revert` 回上一步。

## 顶层设计（The One Idea）

**一切都是"频道"。** 11 个平级频道（♡ Favorites + Lo-fi + Jazz + Blues + R&B + Rock + Pop + Electronic + Classical + Ambient + Folk）串成一条横向序列。

| 场景 | 横滑切换 | 按钮切换（主页 ◁ ▷） |
|---|---|---|
| 主页 | 11 个频道 | 当前频道内的风格 |
| 沉浸页 | 11 个频道 | 无按钮（沉浸纯手势） |
| 详情页 | 11 个频道（每页是该频道的风格列表） | 无 |

**肌肉记忆**：App 内凡是横滑，都是换频道。

## 抓手 1：Evolve 修对

### 问题（基于代码和老鱼反馈）

`AppState.evolve()`（`AppState.swift:393-414`）当前只做：

- `temperature ±0.2`、`guidance ±0.5`、`density ±0.15`、`brightness ±0.15`（config 数值抖动）
- `sendPrompts(prompts)` 原样重发（prompts **一个字没改**）

老鱼听感反馈 1：**差别不大** → Lyria 是 prompt-driven 的，config 扰动远不如 prompt 内容变化明显。
老鱼听感反馈 2：**有时直接切风格** → 根因是并行的 `sessionRotationTimer` 定时硬换台（25 分钟），不是 `evolveTimer` 的锅。

### 设计

Evolve 回归 Phase 2.6 定义（同风格内微调，红线：避免突然风格转变）。**在原 prompts 基础上拼接一组"微调词"**，定时变化。

**微调词池（方向，实装时由小克基于 Lyria 测试细化）**：

- **加/减乐器**：`+ soft piano`、`+ warm pads`、`- drums`、`- bass`
- **能量/密度**：`+ denser`、`+ minimal`、`+ lush`、`+ sparse`
- **纹理**：`+ vinyl crackle`、`+ reverb wash`、`+ tape warmth`

每次 `evolve()` 被触发：
1. 从微调词池随机挑 1-2 条
2. 拼接到当前 style 的 prompts 末尾
3. `lyriaClient.sendPrompts(newPrompts)` 发送
4. config 扰动保留但幅度减半（作为辅助）

**红线**：微调词池不能包含"跨流派"词（比如不能在 Lo-fi 里插"orchestral"），避免听感跳变。

### 影响的文件

- `AppState.swift:393-414` — 重写 `evolve()` 逻辑
- `PromptBuilder.swift` — 新增 `evolveVariant(style:) -> [Prompt]` 工具函数
- 新建 `EvolveVocabulary.swift` — 微调词池分类（通用池 + per-category 池）

## 抓手 2：Auto Tune 设置项（从零新建）

### 问题

v1.1.0 已**删除** `sessionRotation` 相关全部代码（违反"Evolve 不换台"+ "不破坏沉浸感"）。v1.1.1 产品决策：**重新引入但作为用户自选项**，解决深度用户"听久想换"的需求，但不默认打扰。

### 设计

**命名**：暂定 `AUTO TUNE`（Settings 页区段标签）。备选：`CHANNEL DRIFT` / `AUTO ROTATE`。小克实装时用 `AUTO TUNE`，老鱼上 TestFlight 体验后可再改。

**UI**：Settings 页，Evolve section **正下方**并排视觉（同样的 label + Toggle 按钮风格）：

```
EVOLVE                 ⓘ
[Lock] [10 sec] [1 min] [5 min]

AUTO TUNE              ⓘ
[ Toggle ]  Off / On
└─ 默认 Off
└─ ⓘ 文案："Cycle through stations in this channel every 25 minutes to keep the sound fresh."
```

**数据迁移**：
- v1.0 曾有 `sessionRotationEnabled` UserDefaults key，v1.1.0 删除代码后旧 key 仍残留在老用户设备上
- v1.1.1 新实现用**新 key** `autoTuneEnabled`（避免误继承 v1.0 老值）
- 旧 `sessionRotationEnabled` key 在 v1.1.1 启动时**主动清理一次**（UserDefaults `removeObject`），避免脏数据留存

### 影响的文件

- `AppState.swift` — 新增 `autoTuneEnabled: Bool = false`、`autoTuneTimer: Timer?`、`restartAutoTuneTimer()`、`init()` 清理旧 key
- `SettingsView.swift` — 在 evolveSection 后增加 autoTuneSection（Toggle 样式）

## 抓手 3：横滑语义换绑

### 问题

`SpectrumCarouselView.swift:25-39` 的水平滑当前绑定 `VisualizerStyle.allCases`（28 个画风），换的是 `state.selectedVisualizer`。v1.1.1 要把语义换成"换频道"（11 个分类含 Favorites）。

### 设计

**数据源**：新建 `state.channels: [Channel]`，第 0 个固定是 Favorites，后 10 个对应 `StyleCategory.allCases`。

```swift
enum Channel: Hashable {
    case favorites
    case category(StyleCategory)
}
```

**绑定**：`SpectrumCarouselView` 的 `ScrollView` 遍历 `state.channels`，`onChange(of: scrollPosition)` 改为设置 `state.currentChannel`。

**副作用**：
- 切到 `.favorites` 频道时，`currentChannel` 内的风格 = `state.pinnedStyles`
- 切到 `.category(.lofi)` 时，`currentChannel` 内的风格 = `MoodStyle` 预设中 `category == .lofi` 的
- 切换频道时**自动播放该频道首曲**（Favorites 空时保持当前播放不切）

**可视化器在哪**：主页/沉浸页的频谱视觉 = `state.selectedVisualizer`，但**水平滑不再改它**。可视化器选择入口移到 Settings 页（抓手 6）。

### 影响的文件

- 新建 `Channel.swift` — Channel 枚举 + channels 列表工具
- `AppState.swift` — 新增 `currentChannel`、`stylesInCurrentChannel`、`switchToChannel(_:)`
- `SpectrumCarouselView.swift:12,25,37` — `styles` 改为 `channels`，`onChange` 改为 `switchToChannel`
- 沉浸页自动继承（复用 `SpectrumCarouselView`）

## 抓手 4：◁ ▷ 按钮 handler

### 问题

`PlayControlView.swift:10,41` 的 `state.previousStyle() / nextStyle()` 当前实现需要核实是不是"分类内"。老鱼需求：**分类内循环**（Lo-fi Chill → Lo-fi Jazz → Lo-fi Rain → Lo-fi Chill）。

### 设计

在 `AppState` 新增：

```swift
var stylesInCurrentChannel: [MoodStyle] { ... }
func previousStyleInChannel() { ... }
func nextStyleInChannel() { ... }
```

`previousStyle() / nextStyle()` **保留原名不改**（对齐 PlayControlView 调用点），内部实现改为在 `stylesInCurrentChannel` 上循环切换。

**Favorites 频道特殊**：在 `pinnedStyles` 列表内循环。列表为空时按钮置灰（但按 UI 零改动原则，可保持按钮可点击、点击后 no-op，避免视觉变化）。

### 影响的文件

- `AppState.swift` — 改 `previousStyle() / nextStyle()` 实现
- `PlayControlView.swift` — **不改**（按钮和 handler 签名不变）

## 抓手 5：详情页重写

### 现状（基于 `DetailsView.swift`）

目前是 Tab 栏 + 列表结构（`selectedTab: ChannelTab`），老鱼选的新方向是**横滑 Page**。

### 设计

**结构**：

```
┌─────────────────────────┐
│  [MiniPlayer: 风格名 · ♥]│  <- 顶部固定
├─────────────────────────┤
│                         │
│  • • • • ○ • • • •  ←   │  <- pill indicator (11 个点，当前分类高亮)
│                         │
│  [分类名]               │  <- 当前页的频道标签
│                         │
│  ┌─────────────────┐    │
│  │ Lo-fi Chill   ♥ │    │  <- 风格行（可点切台、可点 ♥ 收藏）
│  ├─────────────────┤    │
│  │ Lo-fi Jazz    ♡ │    │
│  ├─────────────────┤    │
│  │ Lo-fi Rain    ♡ │    │
│  └─────────────────┘    │
│                         │
│  横滑 → 下一分类        │
└─────────────────────────┘
```

**TabView + PageTabViewStyle**（SwiftUI）：

```swift
TabView(selection: $state.currentChannel) {
    ForEach(state.channels, id: \.self) { channel in
        ChannelPageView(channel: channel, state: state)
            .tag(channel)
    }
}
.tabViewStyle(.page(indexDisplayMode: .never))
```

pill indicator 自己画（11 个小点 + 当前高亮），样式复用 `SpectrumCarouselView` 的 dots 视觉。

**频道切换联动**：
- 详情页横滑 → `state.currentChannel` 更新 → 主页频谱 carousel 也同步滚动（SwiftUI `@Observable` 自动）
- 反向一样：主页滑频谱 → 详情页当前页同步更新

### 影响的文件

- `DetailsView.swift` — 完全重写
- 新建 `ChannelPageView.swift` — 单个频道页（风格列表）
- 旧 `ChannelTab` 枚举删除（Favorites 改为 `Channel.favorites`）

## 抓手 6：可视化器入口下移

### 现状

`SettingsView.swift:174-197` visualizerSection 当前只是静态文字 `"11 built-in visualizers"`。主页/沉浸页的水平滑换画风入口被抓手 3 占用。

### 设计

visualizerSection 改为**可交互列表**：
- 显示当前选中的 visualizer（有小预览/标题）
- 点击进入子页面 or 弹出 sheet，可选择切换

**颗粒度**：这个抓手优先级低，可用最简实现：

```
VISUALIZERS
┌─────────────────────────┐
│ ≈ Spectrum Bars    →    │  <- 点击 → 下一个 visualizer（循环）
└─────────────────────────┘
```

点击 row 就循环切下一个 visualizer。不做完整选择列表（留 v1.1.2 做可视化器管理）。

### 影响的文件

- `SettingsView.swift:174-197` — visualizerSection 改为可点击循环切换

## 数据流图

```
[SpectrumCarouselView 水平滑]
    ↓
[state.currentChannel = .category(.jazz)]
    ↓
[state.stylesInCurrentChannel 重算 → Jazz 下 4 个 MoodStyle]
    ↓
[首曲自动播放 → lyriaClient.sendPrompts]
    ↓
[DetailsView TabView 自动滚到对应 page（@Observable）]
    ↓
[PlayControlView ◁ ▷ 在 stylesInCurrentChannel 循环]

[evolveTimer 触发]
    ↓
[evolve() 拼接微调词 → 新 prompts]
    ↓
[lyriaClient.sendPrompts（无视觉变化，纯音频微变）]

[sessionRotationTimer 触发（仅当用户 Auto Tune ON）]
    ↓
[nextStyle() → stylesInCurrentChannel 下一首]
    ↓
[后台换台，无 UI 打断]
```

## 错误处理

- **Favorites 频道空**：主页切到 Favorites 时，不自动切台（保持当前播放），pill indicator 正常显示在 ♡ 位置
- **分类内风格空**（理论不会发生，预设每类 3-4 个）：按钮 no-op，不崩
- **Channel 枚举编解码**：`UserDefaults` 持久化 `currentChannel` 时用 `rawValue` 字符串（`"favorites"` / `"category.lofi"`），兼容旧数据（v1.0 没存这个 key，默认 `.category(.lofi)`）

## 测试要点（验收清单）

1. 打开 App → 默认停在 Lo-fi 频道，Lo-fi Chill 播放（首次启动行为不变）
2. 主页左右滑频谱 → 频谱切画面，顶部风格名变为新频道的首曲，音频换台
3. 沉浸页（上滑 Page 0）左右滑频谱 → 同主页联动
4. 详情页（下滑 Page 2）→ pill indicator 高亮当前频道，横滑 → 主页同步
5. 点击 ♥ 收藏风格 → Favorites 频道出现该风格，横滑到 Favorites 能看到
6. Favorites 空时，Favorites 频道页显示 empty state（保留 v1.0 文案）
7. 主页 ◁ ▷ 按钮 → 在当前频道内循环切风格
8. Evolve 设为 10s → 播放 30 秒后能**听到**变化（加/减乐器层），但风格没变
9. Auto Tune 默认 Off，25 分钟**不**自动换台
10. Auto Tune 手动开 → 25 分钟后后台切到当前分类的下一首风格（无 toast、无打断）
11. v1.0→v1.1.1 升级用户 → `pinnedStyles` 保留；老 `sessionRotationEnabled` key 被清理；新 `autoTuneEnabled` 默认 false
12. Settings 页可视化器 row 点击 → 主页频谱画面切换
13. 主页 UI 像素级对比 v1.0 → 无差异（零视觉改动验证）

## Git 提交顺序（对齐可逆原则）
