## 0. 硬约束（高于一切）

1. **不破坏沉浸感**：付费墙**不能在用户正在聆听时打断**、不盖住频谱、不全屏遮罩。任何付费墙必须由**用户主动动作**（点 🔒 / 点 Upgrade 按钮）触发，用 `.sheet(detent: .medium)` 而非 fullscreen
2. **不破坏零引导**：付费墙文案 ≤ 1 屏、不需要解释、自立即能看懂。没有教程式 banner、没有"了解订阅"二级跳转
3. **可逆**：UserDefaults 新 key 设 `sim.v13.` 命名空间；删除分支代码全部行为回 v1.2.1（Flow 全功能解锁）
4. **试用期友好**：v1.3 上线第一周 Tune/Studio 年订享 **50% off**（StoreKit Introductory Offer），老用户红利。不做"前 100 名"（StoreKit 无法做计数上限）

## 4. 风险与回滚

| 风险 | 对策 |
|---|---|
| ASC 订阅审核被拒（最常见原因：产品描述/截图不合规） | P7 提交前用 Apple HIG subscription checklist 自查；被拒后 24h 内改 metadata 重提 |
| StoreKit 2 Transaction.updates 漏事件（renew 状态不同步） | `TestFlight` 跑 3 天专测 renew / cancel / refund 三个场景，用 sandbox account `Settings → Developer → StoreKit` |
| 老用户升级后没看到 Tune/Studio 按钮（缓存产品未刷新） | `StoreKitManager.loadProducts()` 在 App foreground 进入时无条件重跑 |
| Lyria 突然转正价但 UsageLimiter 没开 | P6 feature flag 放在 remote config 或 ASC app review response 里，CEO 在 ASC 后台一键打开（v1.3 binary 已自带） |
| Flow 用户体感退步（v1.2.1 全解锁 → v1.3 收藏变 🔒） | UpgradeView 不激进、Flow 保留完整电台体验（播放/Evolve/Sleep/11 visualizer 不锁） |

**回滚点**：
- 单 commit 级：每个 P 阶段独立 commit，`git revert <sha>` 回滚
- Phase 级：tag `v1.2.1-pre-v1.3-storekit` 整体回 v1.2.1
- 用户级：老用户升级 v1.3 后，Flow 档功能 = v1.2.1 减去「收藏 / 选具体风格 / 拼标签」——如果老鱼觉得太陡，P5 改为「老用户（v1.3 前安装）保留 Tune 档权限 6 个月」迁移补偿
