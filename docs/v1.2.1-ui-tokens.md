## 原则回顾（每条 token 必须扛过这 5 条）

1. 物件感 > UI 感 — 视觉是房间里的一件物，不是图表
2. 沉浸优先 — 不主动打断
3. 克制的温度 — 暖意是点缀，不是底色
4. 侧光而非顶光 — 单侧光源，大半在阴影
5. 动而非跳 — 微动（几像素、几度），不 bar 式猛跳

## 包 #1 — Fog / Morandi 分裂修正

### 病灶

v1.2 已铺 Fog City Nocturne 冷色底（oklch(0.15 0.015 250)），但部分频道卡/ accent / label / warning 还带 Morandi 暖灰（肉粉、米褐、橄榄），夜里看两套色系打架 → **不像深夜，像黄昏将散未散**。

### 治法：锁单冷轴

所有 color token 落在**单一冷色轴上**（hue ∈ [230, 280]），禁止带暖残留（hue ∈ [0, 90] ∪ [330, 360]）。暖意只留给**一个**极小 accent（黄铜 label/ember 提示灯），且占比 ≤ 10%（60-30-10 规则里的 10）。

### Token 表（SwiftUI Color 用 OKLCH → 近似 sRGB 给 Color 构造，原始值锁进 DesignTokens 源文件）

| 名字 | OKLCH | 用途 | 旧值对照 |
|---|---|---|---|
| `--fog-bg-deep` | `oklch(0.13 0.018 252)` | 最底层底色（immersive） | 保留 |
| `--fog-bg-surface` | `oklch(0.18 0.022 250)` | 卡片/面板底 | 替掉任何 `gray.opacity(...)` |
| `--fog-bg-raised` | `oklch(0.22 0.025 248)` | 浮起一级（now-playing 条） | 替掉暖灰 |
| `--fog-line-hairline` | `oklch(0.30 0.015 250 / 0.4)` | 1px 分隔线 | 替掉 `Color.gray` |
| `--fog-text-primary` | `oklch(0.94 0.012 250)` | 主文字 | 替掉 pure white |
| `--fog-text-secondary` | `oklch(0.72 0.020 250)` | 副文字 | 替掉 Morandi 暖灰 |
| `--fog-text-tertiary` | `oklch(0.52 0.025 250)` | 三级/hint | — |
| `--fog-accent-indigo` | `oklch(0.62 0.14 265)` | 主品牌 accent（channel 高亮） | — |
| `--fog-accent-brass` | `oklch(0.72 0.08 72)` | **唯一**暖点缀（ember/烛光 label） | 限定在 evolve/lock 状态灯 |
| `--fog-danger-ember` | `oklch(0.62 0.16 28)` | 错误/超限（复用 brass 偏红） | — |

**禁令**：
- ❌ 任何 hue 在 80–150 段（暖绿/黄绿/橄榄）
- ❌ 任何 hue 在 330–30 段（粉/肉色/米褐），**brass/ember 除外**
- ❌ chroma > 0.06 出现在 bg-* 或 text-* 上（底色不许鲜艳）

**中性向品牌色偏**：所有 neutral 的 chroma 都 ≈ 0.015–0.025（微冷调），不要 0 chroma 纯灰——纯灰配深蓝底会显脏。

**替换范围清单**（iOS Engineer 落地时 grep）：
- `simone ios/Simone/Views/**.swift` 里所有 `Color.gray` / `Color.secondary.opacity` / `Color(white:)` → 映射到上表
- `Assets.xcassets/` 里所有非 fog 系的 color set 审一遍
- SettingsView 的 section header 背景（已知有 Morandi 暖灰）

## 包 #2 — 字号层级加拉开

### 病灶

当前 display/title/body 梯度太近（目测 34/24/17/15），信息不分层，眼睛在一屏里找不到"先看哪"。CEO 另要求：**频道名 +2pt**。

### 治法：5 级 modular scale，ratio 1.3，频道名置于 display-sm

App UI 用固定 rem 级（不用 fluid clamp，产品内 UI 不是 marketing page），字重 & tracking 分层，不止靠尺寸。

### Type Scale（pt，iOS 原生 pt 直接用）

| Token | pt | 字重 | tracking | 字体 | 用途 |
|---|---|---|---|---|---|
| `type-display-lg` | 44 | Unbounded Medium | -0.02em | Unbounded | 沉浸页频道主标题 |
| `type-display-sm` | **28** | Unbounded Medium | -0.015em | Unbounded | **频道卡上的频道名（原 26 → 28，CEO +2pt 落地点）** |
| `type-title` | 20 | Fraunces SemiBold | 0 | Fraunces | section 标题 / 设置分组 |
| `type-body` | 15 | Fraunces Regular | 0.005em | Fraunces | 长文本（setting description） |
| `type-label-caps` | 11 | Archivo Medium | **0.14em** | Archivo | small-caps 标签（LOCK / EVOLVE / BYOK） |
| `type-meta` | 13 | Archivo Regular | 0.03em | Archivo | 时间/计数/副信息 |

**梯度验证**：44 / 28 / 20 / 15 / 13 / 11 → 相邻 ratio ≈ 1.57 / 1.4 / 1.33 / 1.15 / 1.18，满足 impeccable 要求（≥ 1.25）。

**行高**（line-height）：
- display-lg / display-sm：1.1（紧，像标题）
- title：1.3
- body：1.55（深色底的 light 文字需 +0.05–0.1 额外呼吸）
- label-caps / meta：1.4

**光/暗字重修正**：Fog 底色 text-primary 用 Fraunces Regular 看起来比实际瘦，长段落（≥ 4 行）body 升到 Medium。

**line-length 约束**：body 最长 72ch，超过就是 bug（iPad 宽屏也约束到 72ch，居左不铺满）。

**禁令**：
- ❌ 不许 4+ 行 body 段落用 uppercase
- ❌ 不许 display-* 用 gradient-text 填充
- ❌ 不许 title / body 用相同 pt（必须 ≥ 1.3 ratio）

**落地替换点**：
- `Views/Visualizers/*`（沉浸页频道主标题） → `type-display-lg`
- `Views/ContentView.swift`（频道卡上的频道名） → `type-display-sm` 【+2pt 的主落点】
- `Views/SettingsView.swift` section header → `type-title`
- `Views/APIKeySettingsView.swift` 描述 → `type-body`
- 所有 LOCK / EVOLVE / BYOK 徽章 → `type-label-caps`

## 包 #3 — Pause 呼吸动画（不缩图）

### 病灶

v1.2 pause 态下，visualizer 自动缩成 thumbnail（高 ~80pt），原意是"提示已暂停"，但效果是：
1. **打断沉浸**（物件突然变小 → 用户被拽回"我在看 UI" 状态）
2. **层级错位**（大图 → 小图 → 再大图的 context shift 太重）
3. **违反 feedback_no_break_immersion 原则**

### 治法：尺寸不动，用"呼吸速率"表达状态

视觉物本身不动尺寸、不动位置、不动透明度。唯一变化：**生命体征慢下来**——呼吸的节奏从"音乐驱动的连续微动"切到"低频孤立呼吸"，像人睡着了，还在呼吸但不说话。

### Motion Tokens

**播放态（playing）**：
- breathing：`scale 1.000 → 1.006`（6‰，肉眼几乎觉察不到）
- duration：跟随 BPM，默认 8s 一循环（基线）+ spectrum 低频驱动微扰 ±1s
- easing：`cubic-bezier(0.4, 0, 0.2, 1)`（smooth sine-like）
- 叠加：音频驱动的其他微动（几度旋转/几像素偏移）正常走

**暂停态（paused）**：
- breathing：`scale 1.000 → 1.004`（振幅降 1/3）
- duration：**12s 一循环**（慢 1.5x），**固定**，不跟频谱
- easing：`cubic-bezier(0.33, 0, 0.67, 1)`（更慢的 in-out，像深睡呼吸）
- 叠加：音频微动**全部停**，只剩呼吸
- **尺寸 / 位置 / 透明度 / 色彩：0 变化**
- 唯一 UI 层提示：底部 now-playing 条上的 play/pause 图标切换（这个必须有，不然用户不知道状态）

### 过渡（playing ↔ paused）

不要硬切。给一次 1.2s 的缓冲：
- playing → paused：呼吸周期从 8s **渐增**到 12s（3 次呼吸内完成过渡）
- paused → playing：12s **渐减**到 8s（同 3 次呼吸内完成）

过渡期间振幅同步渐变（6‰ ↔ 4‰）。

### NowPlaying artwork 另走一条路

锁屏/灵动岛/控制中心的 artwork 不受此 token 约束——那是系统面板需要的静态缩略图，由 `AudioEngine` 的 `nowPlayingInfo.artwork` 另起一路生成（v1.1.0 已做过类似事，沿用既有路径），**绝不复用 app 内 visualizer 的尺寸状态**。

### 禁令

- ❌ 不许 visualizer 在 paused 时改 `scaleEffect` 缩成 thumbnail
- ❌ 不许 paused 时加 overlay `.gray.opacity()` 蒙版（"压暗提示"是 UI 层的 AI slop）
- ❌ 不许 paused 时出现文字 "Paused"（"温柔陪伴"不告知不必要信息）
- ❌ 不许呼吸周期 < 5s 或 > 20s（太快像焦虑，太慢像死了）

### 落地替换点

- `Views/ImmersiveView.swift`（或 v1.2 的 ImmersivePlayerView，看 iOS Engineer 实际结构）里的 `isPlaying ? fullSize : thumbnailSize` → 删
- `Views/Visualizers/VisualizerModifiers.swift`（或类似 modifier 集中点）新增 `.breathing(state: .playing/.paused)` modifier，封装上述 token
- `AppState.isPlaying` 的 observer：触发 breathing 状态切换（不再触发 size 切换）
